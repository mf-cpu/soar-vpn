# Soar 发版与升级指南

> 这份是给 **你（维护者）** 看的，不给同事看。
> 同事看 `给同事的安装说明.md` 就够了。

---

## 一次性配置（只做一次）

整个升级链路：

```
[mac App 启动 8s 后]
   ↓ GET
http://180.76.134.45:8088/wg-vpn/mac/latest.json   ← 你的 BCC 服务器（小流量）
   ↓ 解析里面的 url
https://github.com/<owner>/<repo>/releases/download/vX/Soar_X.dmg   ← GitHub CDN（大流量）
   ↓ 下载 + sha256 校验 + 自动重启
```

### 多端规划

服务器目录布局已经按多端预留，未来加平台不需要重做服务器：

```
/var/www/wg-vpn/
  ├── mac/          ← 当前 macOS 客户端
  │   └── latest.json
  ├── win/          ← 未来 Windows 客户端
  ├── ios/          ← 未来 iOS
  └── android/      ← 未来 Android
```

每个平台 App 写死自己子路径的 manifest URL，**互相完全独立**：
- mac 的 `DEFAULT_MANIFEST_URL` 在 `src-tauri/src/updater.rs`
- 未来 Windows / 移动端的 App 各自指向自己的 `win/latest.json` 等

加新平台只需要：
1. 在新平台的项目里写死 `http://180.76.134.45:8088/wg-vpn/<plat>/latest.json`
2. 跑该平台的 publish 脚本（参考 `scripts/publish.sh`，把 `BCC_PATH` 和 `PUBLIC_URL` 改成对应子目录）

不需要改服务器、不影响 mac 已部署版本。

为什么这么做：
- 你的 BCC 只有 3Mbps，几十人同时下载 11M 的 DMG 会把别的服务卡死
- GitHub Release 全球 CDN，免费、不限速、不算你流量
- 但 manifest 必须每次发版都更新，所以放在你能 `scp` 上去的地方最方便

### Step 1：在 BCC 服务器上初始化 nginx

```bash
# 在 mac 上把脚本传上去（如果服务器还没 git）
scp scripts/server-init.sh root@180.76.134.45:/tmp/

# ssh 进服务器跑
ssh root@180.76.134.45
bash /tmp/server-init.sh
exit
```

如果你的 BCC 用户名不是 `root`（百度云 BCC Ubuntu 默认 root，CentOS 通常 root，
其它发行版可能是 `ubuntu` / `opc`），把所有 `root@` 换成对应用户。

### Step 2：在百度云控制台放行 8088 端口

控制台 → BCC → 安全组 → 入站规则 → 添加：
- 协议：TCP
- 端口：8088
- 源地址：0.0.0.0/0

### Step 3：在 mac 上验证

```bash
curl http://180.76.134.45:8088/wg-vpn/latest.json
# 应该返回占位 manifest，version 是 "0.0.0"
```

### Step 4：建一个 GitHub repo 用来挂 DMG

可以是 private repo（GitHub Release 的下载链接对所有有 repo 权限的人都是免登录的，
public repo 的下载链接则任何人都能拿）。

- 推荐：建个 **public** repo 叫 `wg-vpn-releases`，里面只放 README，
  让升级链接对所有同事都能直接下载（公司内部用，URL 不外泄就行）。

```bash
# 在你的 wg-vpn 项目目录里
git init
git remote add origin git@github.com:YOUR_NAME/wg-vpn-releases.git
# 不需要 push 代码，repo 只用来挂 release
```

### Step 5：一次性免密配置

```bash
# mac → BCC 免密（避免每次发版要输密码）
ssh-copy-id root@180.76.134.45

# mac → GitHub 免密
brew install gh
gh auth login   # 选 https，按提示来
```

## 每次发版（约 2 分钟）

```bash
# 1. 改版本号
vim package.json src-tauri/tauri.conf.json src-tauri/Cargo.toml
# 把 version 都改成 0.2.4 之类

# 2. 打包
bash scripts/build-mac.sh

# 3. 发版（会让你写 release notes，然后自动上传到 GitHub Release + scp manifest 到 BCC）
bash scripts/publish.sh
```

跑完之后，所有装了 Soar 的同事下次启动 App（最多等 8 秒）就会看到升级横幅，
点「立即升级」全程零交互。

---

## 第一次发版

```bash
# 已经打好包 release/Soar_0.3.1.dmg

# 告诉脚本你的 GitHub repo 在哪
GH_REPO=YOUR_NAME/wg-vpn-releases bash scripts/publish.sh
```

之后只要 `git remote` 配过了，脚本会自动从 git 推断，就不用再传 GH_REPO。

## 排查

| 问题 | 检查 |
|------|------|
| `curl http://180.76.134.45:8088/wg-vpn/mac/latest.json` 超时 | 安全组没放行 8088 |
| 同上，但服务器上 `curl http://127.0.0.1:8088/...` 通 | 同上 |
| App 启动后看不到升级横幅 | 设置 → 关于 → 「检查更新」手动触发，看错误 |
| 升级横幅出现但下载失败 | manifest 里的 `url` 写错了，或 GitHub Release 不是 public |
| 下载完成后安装失败 | 点 App 菜单看日志，多半是 `wg-helper.sh` 没装 sudoers |

## 小流量估算

- 一个 manifest 几百字节，30 个同事每天启动 1~2 次：< 100KB/天
- DMG 11MB，假设每月发 2 个版本，30 人：30 × 11 × 2 = 660MB/月，**全走 GitHub**
- 你的 BCC 流量影响：≈ 0
